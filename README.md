# Engineering Journal

This repository documents technical growth across full-stack development, systems architecture, cloud integration, and AI-assisted software design.

The goal is to capture not only what was built or studied, but why specific architectural decisions were made, what trade-offs were encountered, and how those lessons inform future cloud-focused solutions architecture and integration engineering work.

## Focus Areas

- SquadSync: full-stack team and competition management platform
- Soccer-Subber: explainable lineup and substitution decision engine
- TeamTracker: serverless event and notification proof of concept
- AWS certification path: SAA → DVA → AIF
- Architecture, cloud, DevOps, and AI-assisted development notes

## Structure

Entries are organized by subject first, then classified by metadata inside each entry.

```text
projects/    Entries tied to a specific project, phase, sprint, or implementation effort
topics/      Reusable technical, certification, architecture, and cloud notes
template.md  Standard entry template and naming guidance
```

Use `projects/<project>/` when the entry is primarily about a project decision, phase, sprint, implementation effort, or retrospective.

Use `topics/<topic>/` when the entry captures reusable technical knowledge that is not primarily tied to one project.

The entry metadata identifies the format, such as project retrospective, technical learning note, architecture reflection, decision review, certification note, or troubleshooting note.
