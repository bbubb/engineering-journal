# Topics

Topic entries capture reusable technical knowledge that is not primarily tied to one project.

Use this area for certification notes, architecture concepts, cloud services, DevOps patterns, troubleshooting notes, and reusable technical reflections.

## Naming

Topic entry filenames should be concise, subject-oriented, and not date-prefixed by default.

Examples:

```text
aws/ec2-foundations.md
aws/iam-mental-model.md
ai-assisted-development/codex-skills-vs-workflow-docs.md
architecture/clean-architecture-boundaries.md
```

Dates belong in entry metadata unless the entry is explicitly a chronological log.
