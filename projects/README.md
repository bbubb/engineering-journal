# Projects

Project entries document decisions, phases, sprints, implementation work, and lessons learned within a specific project.

Use this area when the entry is primarily tied to a project rather than a general reusable topic.

## Naming

Project entry filenames should prefer the most specific stable scope that applies:

```text
phase-<n>-sprint-<n>-<topic>.md
phase-<n>-<topic>.md
<topic>.md
```

Use phase and sprint together when both are known and useful for project navigation. Keep issue numbers in metadata unless the entry is specifically about an issue-level postmortem.
